# 微软雅黑字体替换工具

这是一个用于替换 Windows 系统中微软雅黑字体的工具集，允许用户将系统默认的微软雅黑字体替换为其他自定义字体，同时保留完整的回退机制。

## 功能特点

- **字体替换**：将系统中的微软雅黑字体（包括常规、粗体和轻量版本）替换为自定义字体
- **安全备份**：在替换前自动备份原始字体文件和权限设置
- **字体转换**：自动处理字体名称表合并，确保替换后的字体被系统正确识别
- **一键恢复**：提供简单的恢复机制，随时可以还原为原始字体
- **权限管理**：自动处理系统字体文件的所有权和权限设置

## 工作原理

该工具通过以下步骤实现字体替换：

1. **备份原始字体**：将系统中的微软雅黑字体文件（msyh.ttc、msyhl.ttc、msyhbd.ttc）备份到项目目录下的 `msyh-backup` 文件夹
2. **提取字体名称表**：使用 `ttx` 工具从原始字体中提取 name 表信息
3. **合并名称表**：将原始字体的 name 表合并到目标字体中，确保系统能够正确识别
4. **替换系统字体**：替换 Windows 字体目录中的微软雅黑字体文件
5. **更新注册表**：更新系统注册表以确保字体正确加载

## 前置条件

使用本工具前，请确保您的系统已安装以下依赖：

- **Python 3**：推荐 Python 3.6 或更高版本
- **依赖库**：
  - `psutil`：用于查找和终止占用字体文件的进程
  - 安装方式：推荐使用pipenv或者pipx安装
- **字体工具**：
  - **AFDKO**：Adobe Font Development Kit for OpenType
    - 提供 `otc2otf` 和 `otf2otc` 工具，用于处理 TrueType Collection 文件
    - 安装方式：推荐使用pipenv或者pipx安装
  - **fontTools**：
    - 提供 `ttx` 工具，用于处理字体表信息
    - 安装方式：推荐使用pipenv或者pipx安装

## 使用方法

### 准备工作

1. **安装依赖**：确保已安装所有必要的依赖项
2. **准备目标字体**：在项目目录下创建 `target-fonts` 文件夹，并放入以下字体文件：
   - `Regular.ttf`：常规字重版本
   - `Bold.ttf`：粗体字重版本
   - `Light.ttf`：轻量字重版本

### 替换字体

1. **以管理员身份运行**：右键点击 `replace-msyh.py` 并选择「以管理员身份运行」
2. **确认操作**：脚本会自动备份原始字体、转换目标字体、删除原始字体并提示您注销系统
3. **注销系统**：注销后重新登录，微软雅黑字体被删除
4. **重新以管理员身份运行**：键点击 `replace-msyh.py` 并选择「以管理员身份运行」
5. **确认操作**：脚本会将被伪装成微软雅黑的新字体加入系统，并进行一系列兼容操作
6. **注销系统**：注销后重新登录，新字体生效

### 恢复原始字体

1. **以管理员身份运行**：右键点击 `restore-msyh.py` 并选择「以管理员身份运行」
2. **确认操作**：脚本会自动终止占用字体文件的进程并恢复原始字体
3. **注销系统**：注销后重新登录，原始字体将生效

## 注意事项

- **重要提示**：替换系统字体可能会影响系统稳定性和应用程序兼容性，请确保在操作前备份重要数据
- **管理员权限**：脚本必须以管理员身份运行，否则无法修改系统字体文件
- **权限警告**：脚本会修改系统文件的所有权和权限，这是必要的操作
- **进程终止**：恢复脚本会尝试终止占用字体文件的进程，可能会导致未保存的工作丢失
- **系统重启**：在某些情况下，可能需要重启系统而不是仅注销以确保字体更改完全生效

## 故障排除

### 常见问题

1. **字体未生效**：
   - 尝试重启系统而不是仅注销
2. **进程占用错误**：
   - 如果恢复脚本无法删除字体文件，尝试在安全模式下运行恢复脚本
3. **依赖工具找不到**：
   - 确保 AFDKO 和 fontTools 已正确安装并添加到系统 PATH 环境变量
